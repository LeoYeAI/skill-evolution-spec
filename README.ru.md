## Что это такое?

Большинство агентов ИИ — это инструменты без состояния. После каждой сессии они забывают всё. Эта спецификация описывает **систему эволюции навыков**, которая даёт агентам:

- **Постоянная память навыков** — освоенные процедуры сохраняются между сессиями
- **Телеметрия использования** — агент знает, какие навыки он реально использует
- **Управление жизненным циклом (Curator)** — автоматически архивирует мёртвые навыки, предотвращает деградацию
- **Эффективная загрузка токенов** — вставляет в контекст только однострочные сводки, загружает полный контент по запросу

> «Узкое место — не интеллект. Это проектирование дисциплины.» — §8

## Четыре Столпа

| Столп | Функция |
|-------|---------|
| **Файлы навыков** | Обычный текст `SKILL.md` на диске — читаемый, версионируемый, резервируемый |
| **Инструменты R/W** | `skill_view`, `skill_manage`, `skills_list` — «обучение» = запись файлов |
| **Дисциплина системного промпта** | Жёсткие правила о том, когда сохранять и когда обновлять |
| **Curator** | Фоновое управление: дедупликация, архивирование, обрезка |

## Curator — Управление жизненным циклом

Часть, которую большинство реализаций пропускает. Без неё библиотека навыков за 3 месяца превращается в мусорную кучу.

**Автомат состояний:**
```
active ──(stale_after_days)──> stale ──(archive_after_days)──> archived
```

Никогда не удаляет. Худший случай: `archived`. Всегда восстановимо.

**Значения по умолчанию:**
```yaml
curator:
  enabled: true
  interval_hours: 168       # еженедельно
  min_idle_hours: 2
  stale_after_days: 30
  archive_after_days: 90
  backup:
    enabled: true
```

## Контрольный список реализации

- [ ] 1. Формат SKILL.md + схема frontmatter (с полем происхождения `created_by`)
- [ ] 2. Соглашение о директориях + загрузчик при запуске (вставлять только description)
- [ ] 3. Кэш снапшотов (manifest = mtime_ns + size)
- [ ] 4. Инструменты: `skill_view` / `skill_manage` / `skills_list`, атомарная запись
- [ ] 5. Sidecar телеметрии `.usage.json` + 3 типа событий
- [ ] 6. Дисциплина системного промпта
- [ ] 7. Curator: триггер простоя + автомат состояний + значения по умолчанию + резервное копирование + без удаления
- [ ] 8. Логика исключения pin/unpin
- [ ] 9. CLI-команды: status / run / pause / resume / pin / unpin / archive / restore / prune / backup / rollback
